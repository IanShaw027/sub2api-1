# 运维监控模块 - UX 修复路线图（Roadmap）

**生成日期**: 2026-01-05  
**范围**: Admin 运维监控（Ops Dashboard / 错误日志 / 实时指标）  
**依据文档**: `docs/qa/ux-review-report.md`（UX审查）、`docs/qa/product-review-report.md`（产品）、`docs/qa/tech-review-report.md`（技术）、`docs/qa/sre-review-report.md`（SRE）  

---

## 1. 目标与原则

### 目标（Outcome）
1. **可用性优先**：任何网络抖动/大数据量场景下页面不“假死”、不“静默失效”。  
2. **可解释性**：用户能区分“加载中 / 无数据 / 报错 / 离线/重连中”。  
3. **性能体感**：错误日志在 500+ 条数据量下仍可滚动、可操作。  
4. **一致性与可扩展**：把 UX 修复沉淀为可复用模式（状态机、空状态、骨架屏、列表性能策略）。  

### 原则（How）
- **先止血，再提效**：先修 Blocker（可用性/可信性），再修 Critical（效率/探索），再做 Major（适配/无障碍）。  
- **可验收**：每项修复都提供“验收标准 + 观测指标 + 回归点”。  
- **前后端协同**：UX 表面问题往往由数据形态/接口性能/连接稳定性造成，路线图必须显式包含依赖项。

## 2. 总体节奏（Milestones）

> 建议以 1 周为一个迭代里程碑，确保“每周都有可见提升”。

### M0（已完成）：Blocker 止血（1 天内）
- ✅ 骨架屏/首屏加载态：避免首屏空白闪烁  
- ✅ 错误日志虚拟滚动策略：支持 500/1000 pageSize  
- ✅ WebSocket 重连 + 状态可见 + 连接半开检测（stale）  
- ✅ 错误日志筛选条件 URL Query 持久化（支持分享与返回恢复）  

### M1（本周）：Critical 提效（3–5 天）
- 配置表单实时校验（输入期即时反馈 + 禁用非法提交）  
- 图表增强交互（tooltip、缩放/区间选择、图例/导出）  
- 空状态统一组件化（No Data / Filtered Empty / Error / Unauthorized）  
- 搜索/筛选请求防抖（避免输入期触发过密请求）  

### M2（本月）：Major 补齐（1–2 周，按优先级拆分）
- 移动端关键视图适配（最小可用：Header/核心指标/日志列表）  
- 无障碍（WCAG 2.1 A 优先）：键盘可达、ARIA、对比度、焦点管理  
- 色盲友好配色：避免红绿对立作为唯一信息通道  

### M3（持续优化）：体验与竞争力（滚动迭代）
- 错误日志“聚合/分组”能力（同类错误归因、top N、趋势）  
- 历史趋势与自定义时间范围（与产品审查 P0 对齐）  
- 运营/协作入口（帮助中心/文档入口/一键创建告警/共享视图）

## 3. 任务清单（按优先级）

### P0 / Blocker（可用性）
| 任务 | 用户问题 | 方案要点 | 状态 | 代码/说明 |
|---|---|---|---|---|
| 首屏加载骨架屏 | 首次进入空白 2–3s | 首屏默认进入 loading；首屏展示 skeleton；加载完成过渡 | ✅ 已完成 | `frontend/src/views/admin/ops/OpsDashboard.vue` |
| 错误日志大列表卡顿 | 500+ 记录卡顿/不可用 | 提供 200/500/1000 pageSize；启用虚拟列表渲染 | ✅ 已完成 | `frontend/src/views/admin/ops/components/OpsErrorLogTable.vue` |
| WebSocket 断线无感知 | 数据停止更新但用户不知 | 重连策略 + offline/connecting/reconnecting 状态；半开检测触发重连 | ✅ 已完成 | `frontend/src/api/admin/ops.ts` + Header 状态呈现 |

### P1 / Critical（效率与探索）
| 任务 | 用户问题 | 方案要点 | 验收标准（建议） |
|---|---|---|---|
| 配置对话框实时验证 | 错误需提交后才发现 | 表单 rules + blur/input 校验；错误文案可行动；禁用提交按钮 | 无非法提交进入后端；错误提示 1s 内出现 |
| 图表交互增强 | 无 tooltip/缩放，无法钻取 | Tooltip + 区间缩放 + 导出；与筛选联动 | 鼠标/键盘可触达 tooltip；可缩放到任意区间 |
| 空状态统一 | 无数据时“空白” | 明确 4 态：loading/empty/error/unauthorized | 任一态都有明确文案与下一步动作 |
| 搜索/筛选防抖 | 输入期请求过密/体验抖动 | searchText 变更使用 debounce；其他筛选保持即时 | 连续输入不会触发每字请求；停顿后自动刷新结果 |

### P2 / Major（无障碍与适配）
| 任务 | 用户问题 | 方案要点 | 验收标准（建议） |
|---|---|---|---|
| 键盘导航 | Tab/Enter/Esc 不可用 | 对话框焦点陷阱；Esc 关闭；表格行可聚焦 | 全程可用键盘完成核心路径 |
| 对比度（WCAG AA） | 可读性差 | 调整灰度/背景；设计 tokens | 主文本对比度 ≥ 4.5:1 |
| 色盲配色 | 红绿区分失败/成功 | 增加形状/纹理/标签；替换红绿对立 | 色盲模拟下可区分且不依赖颜色 |
| 移动端适配 | 小屏错位/不可用 | 断点布局；图表改为纵向；表格改卡片/虚拟列表 | 375px 宽度可完成查看与筛选 |

## 4. 关键依赖（跨团队）

### Backend / 性能与稳定性（强相关）
这些不是“纯 UX”，但会直接决定体验是否可达成：
- **错误日志查询索引/分页性能**：建议落实 `docs/qa/tech-review-report.md` 的索引与查询优化（尤其分页与过滤）。  
- **WebSocket 连接资源治理**：`docs/qa/tech-review-report.md` 提到的连接泄漏风险需同步修复，否则前端再怎么重连也可能被资源耗尽拖垮。  
- **轮询兜底与接口限流**：在 WS 不可用时，明确 fallback 间隔与压力控制（SRE 风险）。

## 5. 验收与回归（建议清单）

### 回归场景（必须覆盖）
1. 首次进入 Ops 页面：0.1s 内出现 skeleton；加载后内容稳定，不出现“空白闪烁”。  
2. 断网 30s 再恢复：状态从 offline → reconnecting → connected；数据恢复更新。  
3. 错误日志 pageSize=1000：滚动流畅，点击行/重试按钮仍可用。  
4. 无数据/过滤后无数据：明确提示 + 下一步动作（清除筛选/换时间范围）。  

### 指标口径（建议定义并固化）
- FCP/LCP：采集工具与网络条件固定（Lighthouse 或线上 Web Vitals）。  
- 表格性能：定义“首次渲染时间 / 滚动帧率 / 过滤后重绘时间”至少一个可量化指标。  
- WS 可靠性：重连成功率定义（例如 10 次断线中 60s 内恢复的比例）。
